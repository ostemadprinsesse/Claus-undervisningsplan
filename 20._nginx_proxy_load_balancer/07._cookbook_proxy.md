# Start implementing a proxy for you CookBook

Type: Group work
Motivation: This is partly a technical task, and partly a DevOps taks. Both are important in this class. 

Can be used: For your examproject.

---

Implemet a proxy in your cookbook application. Start out simple, make it a "dumb" proxy that just acts as an interface for your application. Start out by running it all on one Azure VM, with the use of Docker Compose. Later you can split out each server on their own VM.

This is a new feature for yur application. It is important to master the technical part of this feature, but equaly important to be DevOps in your approach. 

Remember to document your work. You will need it later during the exam period.
