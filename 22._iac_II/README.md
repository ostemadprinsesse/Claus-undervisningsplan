# IaC - Infrastructure as Code II
Today we continue dealing with the same topic as last time. The outcome should be that your group, in the master branch of your repository has a infrastructure directory with scripts that setup and tear down your infrastructure (VM on Azure). It is also a required outcome that each member of your group can run these scripts, without any agony or fear. 

## Learning Goals

* Setup a version-controlled infrastructure, for faster path to recovery.
* Use the third DevOps principle of **Continual Learning & Experimentation** to ensure that the whole group is up to date with knowledge and the skills needed.
